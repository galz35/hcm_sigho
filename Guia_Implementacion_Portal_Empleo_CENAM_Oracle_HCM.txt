IMPORTANTE - LEER ANTES DE EMPEZAR
------------------------------
Oracle HCM Cloud NO usa WordPress ni Drupal. Tiene su propio editor visual
llamado "Career Site Builder" que funciona de forma similar (arrastrar y soltar,
secciones, columnas, elementos). Esta guia explica como traducir CADA PAGINA
del PDF de diseño a configuraciones dentro de Oracle HCM Cloud.

La herramienta de Oracle usa esta jerarquia:
  Pagina > Seccion > Fila (Row) > Columna > Elemento

Es equivalente a:
  WordPress Page > Section > Row > Column > Widget/Block

REQUISITOS PREVIOS
================================================================================

1. ACCESO AL SISTEMA
   - Rol: Recruiting Administrator (ORA_PER_RECRUITING_ADMINISTRATOR_JOB)
   - Privilegio: IRC_MANAGE_RECRUITING_ADMINISTRATION
   - El modulo de Recruiting debe estar habilitado (Opt-in en Functional Setup Manager)

2. RECURSOS DE DISEÑO QUE NECESITAS EXTRAER DEL PDF
   Antes de empezar, del PDF extrae y prepara:
   
   [ ] Logo de CENAM para desktop (150 x 40 px, formato PNG/SVG)
   [ ] Logo de CENAM para movil (150 x 40 px, formato PNG/SVG)
   [ ] Imagen hero/banner principal (1920 x 600 px minimo)
   [ ] Imagenes de las secciones (cultura, beneficios, etc.)
   [ ] Iconos de categorias de empleo
   [ ] Paleta de colores (anotar codigos HEX de cada color del diseño)
   [ ] Fuentes tipograficas utilizadas
   [ ] Textos de cada seccion
   [ ] Videos (si el diseño incluye videos)

3. SERVIDOR PARA IMAGENES
   CRITICO: Oracle HCM Cloud NO almacena imagenes internamente.
   TODAS las imagenes deben estar alojadas en un servidor web publico
   (CDN, servidor institucional, etc.) sin requerir autenticacion.
   
   Opciones para alojar imagenes:
   - Servidor web institucional de CENAM
   - Servicio de CDN (Azure Blob, AWS S3, Google Cloud Storage)
   - Servidor web interno con acceso publico

4. RUTA PARA ACCEDER AL EDITOR
   Setup and Maintenance
     > Recruiting and Candidate Experience
       > Candidate Experience
         > Career Sites Configuration
           > [Seleccionar sitio] > Edit

PARTE 1 - PAGINA 1 DEL PDF: PAGINA PRINCIPAL (SPLASH PAGE / HOMEPAGE)
================================================================================
Pagina del PDF: 1
Dimensiones: 2812 x 5255 px (pagina larga con scroll)
Tipo en Oracle: SPLASH PAGE

DESCRIPCION:
Esta pagina es la primera impresion que ve el candidato al entrar al portal
de empleo. Es equivalente al "Homepage" de WordPress. Tipicamente incluye:
- Header con logo y navegacion
- Seccion hero con imagen grande y barra de busqueda
- Categorias de empleo
- Vacantes destacadas
- Informacion de la organizacion

COMO IMPLEMENTAR EN ORACLE HCM CLOUD:

Paso 1.1 - Acceder al editor de la Splash Page
   a) Ir a Career Sites Configuration > Edit
   b) En el panel izquierdo, seleccionar "Splash Page"
   c) Se abre el editor visual

Paso 1.2 - Crear la Seccion HERO (Banner Principal)
   a) Hacer clic en "Add Section"
   b) En el menu de acciones (lado derecho de la seccion):
      - Seleccionar "Background Image"
      - Pegar la URL de la imagen hero (del servidor publico)
      - Ajustar opacidad si el diseño tiene overlay oscuro
   c) Agregar una "Row" > seleccionar layout de 1 columna
   d) Dentro de la columna agregar:
      - Elemento "Headline": texto principal (ej: "Unete a CENAM")
      - Elemento "Paragraph": subtitulo descriptivo
      - Elemento "Search Box": OBLIGATORIO en la Splash Page
        (este es el buscador de vacantes)
   e) Ajustar colores del texto a blanco si el fondo es oscuro

Paso 1.3 - Crear Seccion de CATEGORIAS DE EMPLEO
   a) "Add Section" debajo del hero
   b) Agregar "Row" > seleccionar layout de 3 o 4 columnas
   c) En cada columna agregar:
      - Elemento "Image": icono de la categoria
      - Elemento "Headline": nombre (ej: "Investigacion", "Administracion")
      - Elemento "Button": enlace a busqueda filtrada por categoria
   d) Repetir para cada categoria de empleo del diseño

Paso 1.4 - Crear Seccion de VACANTES DESTACADAS
   a) "Add Section"
   b) Agregar "Row" > 1 columna
   c) Insertar elemento "Job List"
      - Este elemento muestra automaticamente las vacantes activas
      - Se puede configurar para mostrar las mas recientes

Paso 1.5 - Crear Seccion "¿POR QUE TRABAJAR EN CENAM?"
   a) "Add Section" con color de fondo segun diseño
   b) Agregar "Row" > 3 columnas
   c) En cada columna:
      - "Image": foto o icono del beneficio
      - "Headline": titulo (ej: "Desarrollo Profesional")
      - "Paragraph": descripcion breve

Paso 1.6 - Crear Seccion de PIE (si aplica contenido antes del footer)
   a) "Add Section"
   b) Agregar contenido de cierre segun el diseño del PDF
      (puede ser un call to action, informacion de contacto, etc.)

PARTE 2 - PAGINA 2 DEL PDF: PAGINA DE RESULTADOS DE BUSQUEDA
================================================================================
Pagina del PDF: 2
Dimensiones: 2812 x 5112 px
Tipo en Oracle: SEARCH RESULTS PAGE

DESCRIPCION:
Esta pagina muestra la lista de vacantes cuando el candidato realiza una
busqueda o selecciona una categoria. Incluye filtros laterales y tarjetas
de vacantes.

COMO IMPLEMENTAR EN ORACLE HCM CLOUD:

Paso 2.1 - Acceder al editor
   a) En el panel izquierdo, seleccionar "Search Results Page"

Paso 2.2 - Configurar los Filtros de Busqueda
   NOTA: Los filtros NO se configuran en el editor visual.
   Se configuran en la pestana "General" del Career Site:
   a) Ir a la pestana "General"
   b) Buscar la seccion "Search Filters"
   c) Activar los filtros deseados:
      [ ] Ubicacion (Location)
      [ ] Categoria de trabajo (Job Category)
      [ ] Tipo de empleo (Employment Type: tiempo completo, medio tiempo)
      [ ] Familia de puesto (Job Family)
      [ ] Fecha de publicacion
      [ ] Organizacion

Paso 2.3 - Personalizar la apariencia
   a) Agregar una seccion superior con:
      - "Headline": "Resultados de Busqueda" o texto del diseño
      - Opcionalmente una imagen de fondo pequeña (sub-hero)
   b) La lista de vacantes se genera automaticamente
   c) Para cambiar el estilo visual de las tarjetas de vacantes,
      usar CSS personalizado (ver PARTE 6)

Paso 2.4 - Configurar orden de resultados
   En la pestana "General":
   - Configurar el orden predeterminado (por fecha, relevancia, etc.)
   - Configurar el numero de resultados por pagina

PARTE 3 - PAGINA 3 DEL PDF: PAGINA DE DETALLE DE VACANTE
================================================================================
Pagina del PDF: 3
Dimensiones: 2812 x 4984 px
Tipo en Oracle: JOB DETAILS PAGE

DESCRIPCION:
Cuando el candidato hace clic en una vacante, llega a esta pagina que muestra
toda la informacion del puesto y el boton "Aplicar Ahora".

COMO IMPLEMENTAR EN ORACLE HCM CLOUD:

Paso 3.1 - Acceder al editor
   a) En el panel izquierdo, seleccionar "Job Details Page"

Paso 3.2 - Elementos automaticos (no requieren configuracion manual)
   Los siguientes se generan automaticamente desde la requisicion:
   - Titulo de la vacante
   - Descripcion del puesto
   - Requisitos y responsabilidades
   - Ubicacion
   - Tipo de empleo
   - Fecha de publicacion
   - Boton "Aplicar Ahora" / "Apply Now"

Paso 3.3 - Agregar contenido adicional segun diseño
   a) ARRIBA del contenido automatico:
      - Agregar seccion con imagen/banner de la organizacion
   b) ABAJO del contenido automatico:
      - Seccion "Sobre CENAM" con descripcion de la organizacion
      - Seccion de beneficios relevantes al puesto
      - Boton/Link a paginas personalizadas (Cultura, Beneficios)
   c) LATERAL (si el diseño lo tiene):
      - Widget de vacantes similares
      - Informacion de contacto de RRHH

Paso 3.4 - Configurar el flujo de aplicacion
   En Setup and Maintenance > Application Flow Configuration:
   - Definir los campos del formulario de aplicacion
   - Activar/desactivar campos opcionales
   - Permitir aplicar sin crear cuenta (reduce friccion)
   - Habilitar carga de CV

PARTE 4 - PAGINA 4 DEL PDF: PAGINA "SOBRE NOSOTROS" / CULTURA
================================================================================
Pagina del PDF: 4
Dimensiones: 2812 x 3798 px
Tipo en Oracle: CUSTOM PAGE

DESCRIPCION:
Pagina personalizada que muestra informacion sobre la organizacion,
su cultura, mision, vision y valores. Es el equivalente a una
"pagina estatica" de WordPress.

COMO IMPLEMENTAR EN ORACLE HCM CLOUD:

Paso 4.1 - Crear la Custom Page
   a) En el editor, ir al menu de paginas
   b) Hacer clic en "Add Custom Page"
   c) Nombre: "Sobre CENAM" o "Cultura Organizacional"
   d) En "Page Name Display for URL": escribir "cultura" 
      (para URL amigable: tudominio.com/cultura)

Paso 4.2 - Diseñar segun el PDF
   a) Seccion 1 - Hero de la pagina:
      - Background image con foto institucional
      - "Headline": "Conoce CENAM"
      - "Paragraph": descripcion breve
   
   b) Seccion 2 - Mision, Vision, Valores:
      - "Row" con 3 columnas
      - Columna 1: "Image" + "Headline" (Mision) + "Paragraph"
      - Columna 2: "Image" + "Headline" (Vision) + "Paragraph"
      - Columna 3: "Image" + "Headline" (Valores) + "Paragraph"
   
   c) Seccion 3 - Galeria o contenido adicional:
      - Fotos de las instalaciones
      - Videos institucionales (YouTube URL)
      - Testimonios de colaboradores
   
   d) Seccion 4 - Call to Action:
      - "Headline": "¿Quieres ser parte?"
      - "Button": enlace a la pagina de busqueda de vacantes

Paso 4.3 - Vincular la pagina
   a) Agregarla al menu de navegacion del header
   b) Agregar botones/links desde la Splash Page
   c) Agregar links desde la Job Details Page

      - Campos requeridos vs opcionales
   
   b) Campos sugeridos para el template:
      [ ] Titulo del puesto
      [ ] Departamento/Area
      [ ] Ubicacion
      [ ] Tipo de empleo (tiempo completo, medio tiempo, temporal)
      [ ] Rango salarial (opcional, segun politica)
      [ ] Descripcion del puesto
      [ ] Responsabilidades
      [ ] Requisitos minimos
      [ ] Requisitos deseables
      [ ] Beneficios
      [ ] Fecha de cierre

Paso 10.2 - CONFIGURAR APROBACIONES
   Ruta: Tools > Transaction Console > Approval Rules
   
   a) Buscar "Approve Job Requisition"
   b) Configurar flujo de aprobacion:
      Nivel 1: Gerente del area solicitante
      Nivel 2: Subdireccion de RRHH
      Nivel 3: Direccion (si aplica)
   c) Definir condiciones especiales si las hay

Paso 10.3 - CONFIGURAR EL PROCESO DE SELECCION (CSP)
   Ruta: Setup and Maintenance
     > Recruiting and Candidate Experience
       > Candidate Job Applications
         > Candidate Selection Process Configuration
   
   a) Crear nuevo proceso: "Proceso Seleccion CENAM"
   b) Tipo: Standard (incluye Oferta y Contratacion)
   c) Configurar las FASES:
   
   FASE 1: NUEVO
   - Estado: Recibido
   - Estado: En Revision
   - Accion automatica: Enviar acuse de recibo por correo
   
   FASE 2: SCREENING
   - Estado: Pre-seleccionado
   - Estado: No Apto
   - Accion: Cuestionario de screening (si aplica)
   
   FASE 3: ENTREVISTA
   - Estado: Programada
   - Estado: Completada
   - Estado: No Aprobado
   - Accion: Notificacion a entrevistadores
   
   FASE 4: EVALUACION
   - Estado: Evaluacion Tecnica
   - Estado: Evaluacion Psicometrica
   - Estado: Aprobado / No Aprobado
   - Accion: Integracion con proveedor de assessments (si aplica)
   
   FASE 5: OFERTA
   - Estado: Borrador
   - Estado: Pendiente Aprobacion
   - Estado: Extendida
   - Estado: Aceptada / Rechazada
   - Accion: Generacion automatica de carta oferta
   
   FASE 6: CONTRATACION (HR)
   - Estado: Pendiente Documentos
   - Estado: Contratado
   - Accion: Crear registro de empleado en Core HR

Paso 10.4 - CONFIGURAR CAMPOS DEL FORMULARIO DE APLICACION
   Ruta: Application Flow Configuration
   
   Campos sugeridos:
   [ ] Nombre completo (obligatorio)
   [ ] Correo electronico (obligatorio)
   [ ] Telefono (obligatorio)
   [ ] CV / Resume (obligatorio)
   [ ] Carta de presentacion (opcional)
   [ ] LinkedIn URL (opcional)
   [ ] Disponibilidad para iniciar (opcional)
   [ ] Pretensiones salariales (segun politica)

1. IMAGENES: Todas DEBEN estar en servidor publico. Oracle NO las almacena.

   Siempre usar Inspector (F12) para verificar clases reales.

   para nuevas funcionalidades del Career Site Builder.

   Para dudas funcionales, consultar Oracle Help Center:
   https://docs.oracle.com/en/cloud/saas/talent-management/farec/

   para tener URLs descriptivas en vez de numeros.
